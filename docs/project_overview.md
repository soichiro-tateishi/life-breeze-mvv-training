# プロジェクト設計書

## 概要

| 項目 | 内容 |
|------|------|
| プロジェクト名 | Life Breeze インターン向けMVV研修 |
| 目的 | インターン生がMVVを「判断基準」として使えるようになる |
| 対象 | Life Breeze インターン生（新規参加者） |
| 形式 | 60分の研修（講義 + ワークショップ） |
| スライド枚数 | 17枚 |
| 正本 | [公式サイト日本語版](https://www.lifebreezehq.com/ja/mvv) |
| スライド生成ツール | Genspark |
| バージョン管理 | GitHub |

---

## 設計方針

### 1. 正本準拠
- MVV・ブランドメッセージは公式サイトの内容のみ使用
- 既存PDF（manager版）との差分は `docs/site_vs_pdf_diff.md` に記録

### 2. インターン向けの最適化
- 出向マネージャー固有の内容は削除（出向前提、中長期戦略、KPI）
- 専門用語を最小限に抑え、平易な表現を使用
- ワークショップは2つ（自分事化 + アクション宣言）に絞る

### 3. 将来の拡張性
- 英語版を作りやすい構造（日英併記テンプレート）
- manager版との分岐は `{audience}` パラメータで対応
- ファイル命名: `mvv_training_{intern|manager}_v{X}_{Y}.md`

### 4. 非エンジニアでも運用可能
- Markdownで完結（特殊ツール不要）
- ファイル数は最小限（11ファイル）
- どこを直せばいいか一目でわかるディレクトリ構造

## 制約条件

- ワークショップは増やさない（2つで固定）
- スライド枚数は15〜18枚の範囲
- 研修時間は60分厳守
- MVVの文言は正本と完全一致させる
- Genspark投入用の統合原稿を別途作成する
