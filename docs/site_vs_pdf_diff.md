# 公式サイト vs 既存PDF 差分分析

> **目的**: 公式サイト（正本）と既存マネージャー向けPDF（2026.02.01版）の差分を整理し、インターン版で何を採用するか判断する。

---

## Mission の差分

| 項目 | 公式サイト（正本） | PDF（manager版） |
|------|------------------|------------------|
| 日本語 | 笑顔と可能性がめぐる未来をつくる。 | 地域の未来を担う挑戦者に、自立と希望の循環を生む仕組みを届ける。 |
| English | Create a future where smiles and possibilities grow. | Deliver a system that creates a cycle of self-reliance and hope for local changemakers. |
| トーン | シンプル・普遍的 | 具体的・事業寄り |

**インターン版の判断**: 正本を採用。シンプルで覚えやすく、インターンにも直感的に理解しやすい。PDFの「仕組み」「循環」は補足説明として活用。

---

## Vision の差分

| 項目 | 公式サイト（正本） | PDF（manager版） |
|------|------------------|------------------|
| 日本語 | 誰もが、自分の時間を力に変え、前へ進める世界をつくる。 | 自由に使える時間が増え、それが学び・仕事・家族・夢に変わる風景がアフリカ各地に広がる。 |
| English | Build a world where people turn time into strength and move forward. | A future where people gain more time they truly own—turning it into learning, work, family, and dreams—spreading across Africa. |
| トーン | 普遍的・グローバル | アフリカ限定・詳細 |

**インターン版の判断**: 正本を採用。「アフリカ各地」はブランド思想スライドで補足。正本はアフリカに限定せず将来の拡張性もある。

---

## Values の差分

### 構造の違い
- **正本**: 5つの行動指針
- **PDF**: 6つの行動指針（各スライドに「合言葉」付き）

### 対応関係

| # | 公式サイト（正本） | 対応するPDF Values |
|---|------------------|-------------------|
| 1 | まず現場に立つ。 | Value 01: 現地で感じて決める (Go see & decide) |
| 2 | 知ろうとし続ける。 | Value 05: 動いた分だけ学ぶ (Learn by doing) + Value 06: 感謝とリスペクト（一部） |
| 3 | 笑顔につながる価値をつくる。 | Value 03: 「できる方法」を探す (Find doable ways)（部分的） |
| 4 | 受けた価値を、次へつなぐ。 | 該当なし（新規） |
| 5 | 利他で考え、続く形にする。 | 該当なし（新規） |
| — | （なし） | Value 02: スピードを止めない (Keep speed) |
| — | （なし） | Value 04: 相手目線で伝える (Communicate at others' eye level) |

### 分析
- PDF Values 02（スピード）と04（相手目線のコミュニケーション）は正本には明示的にない → マネージャー向け行動指針として特化していた
- 正本 Value 4（受けた価値を次へつなぐ）と Value 5（利他で考え、続く形にする）はPDFにはなかった「循環」「持続性」の思想を反映
- 正本はより「思想的・普遍的」、PDFはより「実務的・具体的」

**インターン版の判断**: 正本の5つを採用。PDFの実務的な要素（スピード、コミュニケーション）は、ファシリテーターノートで補足参照できるようにする。

## まとめ: インターン版で使う素材

### 正本から採用
- Mission / Vision / Values（全文）
- ブランド思想フレーズ
- 事業概要（Selfie、展開国）

### PDFから活用（再編集して使用）
- Life Breezeの原点（洗濯問題 → 時間の喪失）
- レンガ職人の寓話
- なぜMVVが必要か（判断基準の必要性）
- 迷ったらこの順番（判断フレームワーク）
- Workshop: 自分事化 + アクション宣言

### 削除（インターンには不要）
- 出向前提の立ち位置
- Missionの具体化（仕組み化×人の自走）
- 中長期の戦い方（Phase 1-3）
- 中期数値目標（KPI）
- 成功の定義
- ケーススタディワーク
